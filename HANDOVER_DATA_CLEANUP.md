# 최종 인계 전 테스트 데이터 초기화

`prepare_handover.py`는 친구에게 시스템을 인계하기 전에 테스트용 개인정보와 운영 이력을 정리하는 스크립트입니다.

## 보존되는 데이터

- 지정한 관리자 계정 1개
- 서비스 목록과 가격
- 샵 설정 및 예약금 설정
- SMS 템플릿
- 데이터베이스 마이그레이션 버전

## 삭제되는 데이터

- 테스트 고객 계정과 고객 메모
- 테스트 직원 계정
- 직원 근무표, 휴무·반차, 담당 서비스 연결
- 모든 예약과 상태 처리 이력
- 결제 기록
- SMS 발송 로그
- 관리자 알림
- 삭제되는 직원의 업로드 프로필 이미지
- 선택한 관리자 외의 추가 관리자 계정

## 안전 절차

먼저 삭제 계획만 확인합니다. 이 단계에서는 데이터가 삭제되지 않습니다.

```bash
python prepare_handover.py
```

관리자 계정이 여러 개면 보존할 계정을 지정합니다.

```bash
python prepare_handover.py --admin-username 관리자아이디
```

출력된 삭제 건수를 확인한 후 실제 초기화를 실행합니다.

```bash
python prepare_handover.py --admin-username 관리자아이디 --execute
```

관리자 계정이 하나뿐이면 `--admin-username`을 생략할 수 있습니다.

```bash
python prepare_handover.py --execute
```

## 자동 백업

실제 삭제 전에 다음 폴더에 데이터베이스 백업이 자동 생성됩니다.

```text
handover_backups/beauty_shop_before_handover_YYYYMMDD_HHMMSS.db
```

삭제 결과가 잘못되었다면 서비스를 중지하고 생성된 백업 파일을 `beauty_shop.db`로 복원합니다.

## 실행 후 확인

1. 관리자 계정으로 로그인합니다.
2. 고객 목록이 0명인지 확인합니다.
3. 직원 목록이 비어 있는지 확인합니다.
4. 예약 캘린더와 타임라인이 비어 있는지 확인합니다.
5. 서비스 목록, 샵 설정, SMS 템플릿이 유지되는지 확인합니다.
6. 친구의 실제 직원 계정과 근무표를 새로 등록합니다.

## 주의

- 운영 중인 실제 고객 데이터가 들어간 서버에서는 함부로 실행하지 마십시오.
- `--execute` 실행 전 반드시 dry run 결과를 확인하십시오.
- `.env`에는 비밀키가 있으므로 친구 외의 사람에게 ZIP으로 공유하지 마십시오.
